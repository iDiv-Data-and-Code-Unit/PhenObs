Collection import

Importing collections from a CSV file

If you would like to import an already collected data, no matter old or new, you have the option to do so. By creating a CSV file that fits the below template, you could import all the collection and records data:

  • 1st column named “Garden” indicating the plant’s main garden's name.
  • 2nd column named “Subgarden” indicating plant’s subgarden's name.
  • 3rd column named “Day” indicating on which day of the month data was collected for the plant.
  • 4th column named “Month” indicating on which month of the year data was collected for the plant.
  • 5th column named “Year” indicating in which year data was collected for the plant.
  • 6th column named “Species” indicating the plant’s name.

Next column names indicate the what data was collected and acceptable values are: yes, no, unsure, missed

  • 7th column named “Initial vegetative growth”.
  • 8th column named “Young leaves unfolding”.
  • 9th column named “Flowers opening”.
  • 10th column named “Peak flowering”.
  • 12th column named “Ripe fruits”.
  • 13th column named “Senescence”.

Next column names indicate the intensity and acceptable values are: 5%, 10%, 15% ... 95%, 100%

  • 11th column named “Flowering intensity”.
  • 14th column named “Senescence intensity”.

Remaining 2 columns accept the indicated respective values.

  • 15th column named “Maintenance” indicating what maintenance options were noted for the plant. Acceptable values are comma seperated combination of options below:
    • cut_partly
    • cut_total
    • covered_natural
    • covered_artificial
    • transplanted
    • removed
    Example value: cut_partly, covered_natural
  • 16th column named “Remarks” which is a text for collector's note on the plant for the collection.

We advise you to use data entry template for Excel which gives you acceptable options for some of the fields and runs checks on values. However, it is not perfect and is prone to human error. Please double check the entered values, names and dates. Typos may result in corrupt data. Additional checks are run on the server too, nevertheless wrong data entry that fits the above guidelines will be saved and may end up corrupting the data.

You may download the template by clicking on this link.

After the data is entered in Excel, please export the file as CSV before uploading. Make sure to save it as UTF-8 CSV.

Be aware, each subgarden needs to be created prior to uploading the CSV file, hence the plants. If the mentioned subgardens or plants are not found in the system, you will get an error.
